# REILLY-OSINT | Доповнення та розширення архітектури
# Статус: Беклог для Блоку №4 (Аналітика) та Блоку №2 (Збір)
# Гранична довжина рядка: 100 символів

Цей документ містить завдання на розвиток системи, побудовані на стику формальної
логіки, когнітивної психології та методів структурувального аналізу розвідки (SAT).
Реалізується після стабілізації Блоку №2 (Збір через Bright Data).

---

## 🧠 1. Контур боротьби з упередженням підтвердження (Confirmation Bias)
* **Проблема:** Ризик збору ШІ-агентами лише тих фактів, що підтверджують початкову гіпотезу.
* **Задача для Блоку №2 (Збір):** Модифікувати модуль `SearchTypeMapper`. При генерації 
  пошукових dorks для Bright Data система зобов'язана створювати дзеркальні контр-запити.
* **Приклад:** - Прямий пошук: "Ознаки дефіциту та кризи на заводі X"
  - Контр-пошук (Red Teaming): "Ознаки модернізації, розширення та інвестицій на заводі X"

## ⚓ 2. Модуль де-анкорингу (Руйнування «Ефекту якоря»)
* **Проблема:** Перша знайдена в медіа цифра або гучна заява стає для ШІ помилковим базовим 
  орієнтиром (якорем), що викривляє подальший аналіз дельт.
* **Задача для Блоку №4 (Аналітика):** Заборонити ШІ робити висновки на основі поодиноких 
  поточних повідомлень. Запровадити обов'язкове вирахування математичної норми об'єкта 
  (Baseline) за попередні історичні періоди. Поточні дані оцінювати виключно як відхилення 
  (Дельту) від цієї норми.

## ⚖️ 3. Логічний затвор (Захист від галюцинацій та логічних зламів)
* **Проблема:** Великі мовні моделі (LLM) схильні до внутрішніх протиріч у великих звітах.
* **Задача для Блоку №5 (Завершення):** Перед видачею фінального звіту користувачу прогнати 
  текст через жорсткий фільтр формальної логіки (Логічний затвор). 
* **Критерій перевірки (Закон суперечності):** Два протилежні судження про один об'єкт у 
  фізичному світі не можуть бути істинними одночасно (наприклад, "завод стоїть через дефіцит 
  кадрів" та "завод подвоїв випуск"). У разі детекції суперечності — звіт блокується і 
  повертається агентам на перерахунок.
